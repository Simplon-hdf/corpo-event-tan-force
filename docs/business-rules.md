## Règles de gestion

### Utilisateurs

- R1: L'application a plusieurs types d'utilisateurs
- R2: L'utilisateur peut être un admin
- R3: L'utilisateur peut être un collaborateur
- R4: Un admin peut créer un évenement
- R5: Un admin peut lire un évenement
- R6: Un admin peut modifier un événement
- R7: Un admin peut supprimer un événement
- R8: Un admin peut créer un commentaire
- R9: Un admin peut lire un commentaire
- R10: Un admin peut modifier un commentaire
- R11: Un admin peut supprimer un commentaire
- R12: Un collaborateur peut lire un événement
- R13: Un collaborateur peut créer un commentaire
- R14: Un collaborateur peut lire un commentaire
- R15: Un collaborateur peut modifier un commentaire
- R16: Un collaborateur peut supprimer un commentaire

### évenements

- R17: Un événement doit avoir un intitulé
- R18: Un évenement doit avoir une date
- R19: Un événement doit avoir une description
- R20: Un événement doit avoir un auteur
- R21: Un evenement peut avoir un ou des commentaires

### Commentaires

- R22: Un commentaire doit avoir un auteur
- R23: Un commentaire doit avoir un texte
- R24: Un commentaire doit avoir une date
